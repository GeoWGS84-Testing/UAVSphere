# 🔵 Test Case 2.0 → 2.6

(WorldView01, 02, 03, 04, GeoEye1, QuickBird, IKONOS)

These use **full validation mode**.

---

## 🟢 OUTLINE Validation

### What it does:

* Clicks Outline button.
* Waits up to ~90 seconds for map overlay.

### What it checks:

* Overlay detected via:

  * `<img>` containing sceneId
  * OR SVG path
  * OR canvas layer
* Extracts bounding box.
* Saves outline JSON.
* Takes screenshot after stabilization.

### Failure conditions:

* Overlay not found → warning + screenshot.
* No bounding box → warning.

---

## 🟢 PREVIEW Validation (Strict)

### What it does:

* Click Preview.
* Waits for preview overlay.

### What it checks:

1. Image exists.
2. Image has valid `src`.
3. Image loads successfully.
4. Bounding box overlap with outline:

   * Uses `bboxIntersects(...)`
   * Minimum overlap ratio = 0.05
5. Saves screenshot.

### Failure conditions:

* No overlay → warning.
* No overlap with outline → warning.
* Image missing → warning.

---

## 🟢 DETAILS Validation (Strict)

### What it does:

* Click Details.
* Waits for modal and image (up to 120s).

### What it checks:

1. Modal visible.
2. Detail image visible.
3. Image `src` contains `sceneId`.
4. Preview filename matches detail filename.
5. Details table visible (`#tbl_details`).
6. Table contains expected fields:

   * id
   * bbox
   * properties
   * assets

### Failure conditions:

* Image not loaded → warning.
* Filename mismatch → warning.
* Missing table data → warning.

---
